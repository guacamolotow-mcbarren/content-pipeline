# Style Guide: Статьи для СМИ

> Эталон: «ИИ-агенты для управленца: три способа сэкономить 12 часов в неделю»
> Фрагменты: [references/article-vasilchenko.md](../references/article-vasilchenko.md)

## Формат
- Экспертная статья-практикум для любого СМИ (деловое, отраслевое, образовательное)
- How-to с готовыми промптами и сценариями
- Длина: 10000–18000 знаков
- Требования площадки (рубрика, SEO, формат заголовка) — уточнять в брифе

## Тон
- Образовательный, практический, без занудства
- Обращение к целевой аудитории из брифа («руководитель», «менеджер», «специалист»)
- Допустимы цифры из исследований, ссылки на внешние источники
- Баланс: проблема → решение → эффект → риски

## Структура

```markdown
# [Заголовок: польза + цифра/конкретика]

[Лид: 2–3 абзаца — проблема, контекст, что получит читатель]

[Опционально: блок о рубрике / серии материалов — если требует площадка]

## [H2 — сценарий / блок 1: конкретное название]

**Проблема**
[1–3 абзаца: боль аудитории с примером]

**Подготовительный этап**
- [чеклист из 2–4 пунктов]

**Шаблон запроса**
[Готовый промпт для ИИ-агента — блок текста]

**Что делает ИИ**
[1–2 абзаца: что агент делает автономно]

**Практический эффект**
[Конкретный результат: «15–20 минут вместо нескольких дней»]

**На что обратить внимание**
[1–3 пункта: риски, NDA, безопасность]

## [H2 — сценарий / блок 2]
...

## [H2 — вставной блок с данными / контекстом]
[Статистика, исследование — усиливает аргументацию]

[Финальный абзац: резюме + призыв масштабировать]
```

## Паттерны из эталона

### Заголовок
- Формула: `[Тема]: [количество] способов [результат с цифрой]`
- Пример: «ИИ-агенты для управленца: три способа сэкономить 12 часов в неделю»

### Лид
- Первое предложение — контраст: «Большинство X используют Y только для Z, но…»
- Второе — ключевой инсайт: «эффективность зависит не от промптов, а от сценариев»
- Третье — обещание: «Разберём три сценария…»

### Сценарии (повторяющийся шаблон)
Каждый сценарий — **одинаковая структура из 6 блоков**:
1. **Проблема** — боль + пример из жизни целевой аудитории
2. **Подготовительный этап** — маркированный список
3. **Шаблон запроса** — готовый промпт (можно копировать)
4. **Что делает ИИ** — описание автономных действий
5. **Практический эффект** — цифра экономии времени
6. **На что обратить внимание** — риски (NDA, API-ключи, галлюцинации)

### Промпты в тексте
- Начинать с роли: «Ты — аналитик данных» / «Ты — технический архитектор и бизнес-аналитик»
- Нумерованные шаги: «1. Подключись… 2. Выгрузи… 3. Для каждого…»
- Конкретные плейсхолдеры: `[JIRA_URL]`, `[API_TOKEN]`, `[PROJECT_KEY]`
- Завершать форматом выхода: «Сформируй HTML-отчёт…» / «Подготовь executive summary…»

### Вставные H2 между сценариями
- Статистика для усиления: «68% пользователей экономят втрое меньше…»
- Объяснение термина: «Чем ИИ-агент отличается от генеративного ИИ»

## Оформление
- **H2** для сценариев и крупных блоков (не H3)
- **Жирный** для меток блоков: **Проблема**, **Шаблон запроса**, **Что делает ИИ**
- **Списки** `-` для подготовительного этапа и рисков
- **Промпты** — plain text, без code fence (или с fence для длинных)
- **Цифры** — конкретные: «12 часов в неделю», «6% пользователей», «68%»

## Обязательные элементы
- Автор в метаданных (имя + «Инфосистемы Джет») — не обязательно в теле
- Минимум 2–3 пронумерованных сценария с одинаковой структурой
- Минимум 1 готовый промпт-шаблон
- Минимум 1 блок статистики / исследования
- Блок «На что обратить внимание» в каждом сценарии (риски!)
- Финальный абзац с резюме

## Лексика
- Целевая аудитория — по брифу («руководитель», «управленец», «менеджер» и т.д.)
- «ИИ-агент», «агентский режим», «промпт» — без упрощения
- «Практический эффект» — ключевая метка
- Избегать: код на Python/JS, технические листинги (это для категории `article-tech`)

## Запреты
- Статья без готовых промптов / шаблонов
- Сценарии без блока рисков
- Абстрактные советы без цифр эффекта
- Кликбейт в заголовке без содержания
- Технический deep dive (переносить в категорию `article-tech`)
