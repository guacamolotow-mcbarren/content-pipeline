# Style Guide: Интервью для СМИ

> Эталон: интервью с Андреем Янкиным (дек 2025)
> Фрагменты: [references/interview-yankin.md](../references/interview-yankin.md)

## Формат
- Журналистское интервью с экспертом(ами) «Инфосистемы Джет» для любого СМИ
- Формат «вопрос — ответ», вопросы от лица журналиста
- Длина: 8000–15000 знаков (10–20 блоков Q&A)
- Требования площадки (объём, подзаголовки, SEO) — уточнять в брифе

## Тон
- Экспертный диалог, без рекламного пафоса
- Спикер говорит живо: допустимы аналогии, ирония, риторические вопросы
- Журналист задаёт уточняющие и провокационные вопросы («А как же ИИ?», «Неужели не очевидно?»)
- Цифры и кейсы из практики, не абстракции
- Уровень терминологии — по аудитории из брифа (отраслевое СМИ / массовое / деловое)

## Структура

```markdown
# [Имя], «Инфосистемы Джет»: [Заголовок-тезис через двоеточие]

[Лид-абзац: 2–3 предложения — контекст темы + кто говорит + что узнает читатель]

[Опционально: короткая врезка-цитата спикера перед первым вопросом]

[Вопрос журналиста — развёрнутый, с контекстом]

[Имя спикера]: [Ответ — 2–8 абзацев, с примерами]

[Следующий вопрос...]
```

## Паттерны вопросов (из эталона)

| Тип | Пример |
|-----|--------|
| Контекст рынка | «Как сегодня меняется характер киберугроз…?» |
| Уточнение / challenge | «А как же искусственный интеллект? Все говорят про бот-фермы…» |
| Практика | «Какие технологии реально повышают способность восстанавливаться?» |
| Скепсис | «Часто сканер выдает 10–15 тысяч "красных" пунктов. Что делать?» |
| Финальный | «Какую главную мысль вы бы хотели донести до читателей?» |

## Паттерны ответов спикера

- Начинать с факта или тезиса, не с «конечно» / «безусловно»
- Конкретные цифры: «80% инцидентов», «38 тысяч рублей», «список из 80 систем»
- Кейсы: «буквально на днях расследовали…», «недавно работали с компанией…»
- Аналогии: «бэкапы — как подушка безопасности в автомобиле»
- Списки через «Ещё важны:» с маркерами или нумерацией
- Признание сложности: «перестроиться невероятно тяжело», «универсального ответа нет»

## Оформление

- **Спикер:** «Андрей Янкин:» (имя + двоеточие, без «— сказал»)
- **Журналист:** вопрос без атрибуции или с контекстом в тексте
- **Цитаты внутри ответа:** без кавычек, живой разговорный стиль
- **Аббревиатуры:** по уровню аудитории из брифа (для отраслевой — без расшифровки; для массовой — с пояснением при первом упоминании)
- **Абзацы:** 2–5 предложений, чередование коротких и развёрнутых

## Обязательные элементы
- Заголовок: `[Имя], «Инфосистемы Джет»: [сильный тезис]`
- Лид с полным именем и должностью спикера
- Минимум 3 конкретных кейса или цифры
- Минимум 1 провокационный / уточняющий вопрос от журналиста
- Финальный вопрос «главная мысль»
- 8–15 пар вопрос-ответ

## Запреты
- Монологи без вопросов (>3 абзаца подряд от спикера без Q)
- «Уникальный», «революционный», «лучший на рынке»
- Прямые призывы к покупке услуг Jet
- Выдуманные цифры и кейсы

## Мульти-спикер (если 2–3 эксперта)
- Чередовать спикеров в ответах: «Андрей Янкин: …» / «Юрий Семенюков: …»
- Вопросы адресованы группе: «расскажите, как…»
- Лид перечисляет всех спикеров с должностями
